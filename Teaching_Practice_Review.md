# Teaching Practice Review

## Review Basis

This review checks the Studio Refresh package against widely used university teaching-design principles:

- **Constructive alignment:** learning objectives, activities, and assessments should reinforce one another. Carnegie Mellon's Eberly Center frames this as alignment among objectives, assessments, and instructional strategies.
- **Active learning:** students learn more when they work on tasks, discuss, make decisions, and receive feedback rather than mainly listening.
- **Universal Design for Learning:** CAST's UDL Guidelines 3.0 emphasise multiple means of engagement, representation, and action/expression, with the goal of learner agency.
- **Authentic assessment:** students should produce work resembling the professional practice the course claims to teach.
- **Scaffolding and cognitive load:** complex work should be broken into bounded tasks with visible goals, supports, and feedback.

Reference links:

- CAST UDL Guidelines 3.0: https://udlguidelines.cast.org/
- Carnegie Mellon Eberly Center, assessment/objective/instruction alignment: https://www.cmu.edu/teaching/assessment/basics/alignment.html
- Freeman et al. active learning meta-analysis, PNAS 2014: https://www.pnas.org/doi/10.1073/pnas.1319030111
- Chi & Wylie ICAP framework, Educational Psychologist 2014: https://doi.org/10.1080/00461520.2014.965823

## Verdict

The Studio Refresh is a strong improvement over a lecture-heavy or framework-first course. It is engaging, practical, and authentic: every week asks students to build a useful analytics automation artefact, expose failure, and defend a decision with evidence.

The main risk is that the studio design can feel too open-ended unless instructors enforce structure. The refresh therefore needs explicit alignment, accessibility supports, and assessment rubrics. These have been added in the current package.

## Strengths

| Practice | Evidence in the Studio Refresh |
|---|---|
| Active learning | Each week includes studio sprints, peer demos, failure cards, and exit tickets. |
| Authentic assessment | Artefacts resemble real analytics work: work orders, reconciliation reports, leakage audits, APIs, analyst-agent evals, launch checklists. |
| Feedback-rich design | Students receive feedback from tests, failure cards, peer demo questions, and defence notes. |
| Cognitive apprenticeship | The instructor acts as studio lead, modelling how to scope, break, repair, and defend agent work. |
| Transfer | The repeated Build / Break / Defend pattern transfers across reporting, data quality, modelling, deployment, analyst agents, and launch review. |

## Risks Found

| Risk | Why it matters | Revamp applied |
|---|---|---|
| Alignment could be implicit | Students may not see how studio tasks map to marks. | Added `Portfolio_and_Assessment_Model.md` with weekly bundles and rubric. |
| Case detail could be too thin | Students may understand the mission but not the stakeholder context, data traps, expected evidence, or instructor solution logic. | Added `Weekly_Case_Packs.md` and expanded all six weekly exercise sheets with case setup, learning outcomes, checkpoints, evidence bundles, and fallback routes. |
| Novices may be overwhelmed | Open studio tasks can punish students with weaker coding confidence. | Added facilitation guidance for fallback roles, supplied artefacts, and manual review paths. |
| UDL/accessibility was under-specified | Students need multiple ways to engage and demonstrate competence. | Added inclusion and accessibility guidance in the facilitation guide. |
| Reflection could be too lightweight | Exit tickets are useful only if they feed the portfolio. | Clarified defence notes and viva prompts as portfolio evidence. |
| Instructor variability | A studio course depends heavily on facilitation quality. | Added standard 180-minute seminar shape, failure cards, and instructor checklists. |

## Best-Practice Mapping

| Principle | Studio Refresh implementation |
|---|---|
| Constructive alignment | Weekly mission -> artefact -> failure evidence -> defence note -> rubric. |
| Active learning | Most class time is building, breaking, peer reviewing, and defending. |
| ICAP-style engagement | Students move beyond passive listening into constructive and interactive work: generating artefacts, critiquing peer work, and explaining decisions. |
| UDL engagement | Business missions create relevance; teams have roles; students choose implementation details within a bounded mission. |
| UDL representation | Concepts are delivered through scene, work order, slide, artefact, and failure card. |
| UDL action/expression | Students can demonstrate learning through code, tests, reports, demos, and defence notes. |
| Feedback | Tests, peer questions, instructor defect wall, and exit tickets create repeated feedback loops. |
| Scaffolding | Every mission includes work order, sprint tasks, failure cards, repair moves, evidence checklist, and stretch options. |

## Recommendation

Use the Studio Refresh as the primary teaching package if the goal is practical capability and student engagement. Keep the older framework-heavy materials as instructor reference or optional reading, not as the main classroom flow.
